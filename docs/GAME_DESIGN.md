# GAME DESIGN

## 初期版の範囲

初期版はKING DEIGOのみを使う。機種選択画面、未解放機種、準備中機種のUIは作らない。

新規ゲームは、入店時間選択、10台生成、1台目確認、履歴打ち開始の順に進む。保存データがある場合だけ、続きから、最初からを選べる開始画面を表示する。

## 遊技の流れ

プレイヤーは16時から22時の間に入店し、10台を前後移動で確認する。台移動と履歴確認ではゲーム数を消費しない。

勝負するを押すと1Gずつ内部抽選し、BIG/REG当選、50枚区切りの停止範囲到達、閉店到達のいずれかで停止する。ボーナス後に自動続行はしない。

## 未確定の扱い

入店時履歴生成の回転数分布と空き台補正は、確定値ではなく設定化した調整値として扱う。将来変更できるよう `hall-config.js` に分離する。
