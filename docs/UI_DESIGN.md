# UI DESIGN

## 画面

1. 開始・保存データ確認画面
2. 入店時間選択画面
3. 台確認画面
4. 1勝負の結果表示
5. 退店確認画面
6. 最終結果画面

機種選択画面は作らない。

## 台確認画面

常に現在時刻、残りゲーム数、持ちコイン、総投資、総ゲーム数を表示する。

台番号、総回転数、BIG、REG、合成確率、現在差枚、スランプグラフ、自分の遊技結果、子役記録を確認できる。

操作はHTMLボタンで実装し、背景画像や装飾画像に依存させない。

区切り到達やボーナス当選後は専用結果画面へ移動せず、台確認画面の上部に短い結果通知を表示する。退店確認画面と最終結果画面は残す。

遊技中は「総投資」「持ちコイン」「現在収支」を中心に表示し、台の現在差枚とプレイヤー収支を混同しない。

## 画像差し替え

画像はCSSカスタムプロパティと `asset-config.js` に集約する。画像が存在しない場合もCSSの色とグラデーションで操作できる。
